逻辑属性

网页上的所有东西都是一个盒子，CSS总是使用顶部、底部、左侧和右侧的物理方向来指示我们的目标盒子的哪一边。但是，当writing-mode不是默认的从上到下的水平方向时，这些值往往会令人困惑。

CSS 逻辑属性与逻辑值这一 CSS 模块引入了逻辑属性和逻辑值，由此得以通过逻辑的——而不是实体的——方向与尺度映射控制布局。

块向与行向

逻辑属性和逻辑值用抽象术语块向和行向描述其流向。这些术语的实体含义取决于书写模式。

块向尺度

与行内文本流向垂直的方向上的尺度，如横排书写模式中的竖直尺度，以及竖排书写模式中的水平尺度。对于标准的英文文本，此尺度指竖直尺度。

行向尺度

与行内文本流向平行的方向上的尺度，如横排书写模式中的水平尺度，以及横排书写模式中的竖直尺度。对于标准的英文文本，此尺度指水平尺度。

举例来说

border-block

文字水平方向，border-block是上下方向
文字垂直方向，border-block是左右方向

border-inline

文字水平方向，border-inline是左右方向
文字垂直方向，border-inline是上下方向

边界的物理方向和逻辑方向的比较

给定了一个块级框，里面有一些文本内容，有下面的一些规范：

  1. 文本顶部边框的颜色应该是红色(#FF6347;/*提香红*/)
  2. 文本右边边框的颜色应该是绿色(#2e8b57;/*马尔斯绿*/)
  3. 文本底部边框的颜色应该是蓝色(#00a0b0;/*蒂芙尼蓝*/)
  4. 文本左边边框的颜色应该是黄色(#F9DC24;/*申布伦黄*/)

每次写入方向更改时，使用物理方向都需要进行修改，而使用逻辑属性则不需要
